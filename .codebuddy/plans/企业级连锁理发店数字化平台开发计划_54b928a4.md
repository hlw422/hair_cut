## 产品概述

一套面向全国连锁理发店的完整数字化管理平台，类似「美团丽人 + 连锁门店管理 + 理发师工作台 + 企业ERP后台」的SaaS系统。包含6大子系统：用户端小程序、理发师端小程序、店长端小程序、总部运营后台（Web）、官网宣传网站（Web）、统一API服务端。

## 核心功能

### 用户端小程序

首页（城市/推荐门店/理发师/热门服务/优惠活动/会员专区）、门店列表（距离/评分/价格排序+地图模式）、门店详情（照片/营业时间/评分/评价/导航）、理发师详情（头像/从业年限/擅长风格/作品集/价格/粉丝数）、在线预约流程（选门店→理发师→服务→时间→支付）、订单管理（7种状态）、在线支付（微信支付/优惠券/会员折扣/余额/积分抵扣）、5级会员体系、多类型优惠券系统、积分商城、用户评价（图片/视频/文字/评分）、AI发型推荐（上传照片分析脸型/发质推荐发型）、消息中心

### 理发师端小程序

工作台（今日预约/收入/待服务客户/评价数）、预约管理（今日/未来/历史预约）、客户管理（档案/消费记录/偏好/发型记录）、收入统计（日/周/月/年）、排班管理（请假/调班/排班/休息）、作品管理（图片/视频/案例）、粉丝管理（关注/互动/消息）

### 店长端小程序

门店总览（营业额/客流量/订单量/会员增长）、员工管理（理发师/助理/前台/考勤）、全店预约管理与资源调度、财务管理（收入/退款/提成/成本统计）、库存管理（洗护用品/工具设备）、采购管理（申请/审批/供应商）

### 总部运营后台（Web）

组织架构（集团→区域→城市→门店）、全国门店管理（新增/迁移/停业/审核）、用户管理（会员/标签/黑名单/画像）、理发师管理（认证审核/等级/绩效/培训）、营销中心（优惠券/满减/拼团/秒杀/会员日）、CRM系统（生命周期/分层/流失预警/召回）、财务中心（门店/区域/集团收入/利润分析）、数据分析实时大屏（GMV/客单价/复购率/留存率/转化率）

### 官网（Web）

品牌介绍/故事/文化、全国门店地图查询、明星理发师展示与预约入口、新闻中心（企业新闻/活动资讯/门店动态）、招聘中心、加盟系统（申请/咨询/政策/投资分析）

### 技术能力

腾讯地图集成（附近门店/距离计算/导航/城市切换）、RBAC权限体系（7种角色：超级管理员/总部管理员/区域经理/店长/理发师/客服/会员用户，支持菜单/按钮/数据/门店/区域权限）

## 系统架构

### 整体架构（分层微服务）

采用**单体模块化 + 异步解耦**架构，第一阶段以单体Gin应用内按领域分包，通过RabbitMQ/Redis实现服务间解耦，后续可平滑拆分为独立微服务。

```
客户端层（6个前端应用）
    ↓ HTTP/WebSocket/API Gateway
API服务层（Golang Gin - 按领域模块划分）
├── 用户模块 (user)
├── 门店模块 (store)
├── 理发师模块 (stylist)
├── 预约订单模块 (appointment/order)
├── 支付模块 (payment)
├── 会员积分模块 (membership/points)
├── 优惠券模块 (coupon)
├── 营销活动模块 (marketing/campaign)
├── 消息通知模块 (notification/message)
├── 内容模块 (content/review/portfolio) - 含AI发型推荐
├── CRM模块
├── 财务模块 (finance)
├── 库存采购模块 (inventory/purchase)
├── 组织人事模块 (org/hr)
├── 数据分析模块 (analytics)
└── 系统管理模块 (system/auth/rbac)
    ↓
基础设施层
├── MySQL (主业务数据)
├── Redis (缓存/Session/分布式锁/排行榜)
├── Elasticsearch (门店/理发师/服务全文搜索)
├── MongoDB (日志/消息/作品媒体元数据)
├── RabbitMQ (异步任务: 支付回调/消息推送/数据同步/搜索索引)
├── MinIO (文件存储: 头像/作品图/门店照片)
└── WebSocket (实时通知: 预约变更/新订单/消息)
```

### 关键技术决策

1. **Taro而非原生微信小程序**: 三端小程序共享业务逻辑代码和UI组件库（TDesign），复用率达70%+
2. **单体模块化起步**: 团队规模可控时单体架构开发效率最高，Gin按领域分包，通过interface抽象Repository和Service层，拆分时只需将包移到独立进程
3. **MySQL主库 + MongoDB辅助**: 核心事务性数据走MySQL，日志/消息/非结构化数据走MongoDB
4. **Elasticsearch仅用于搜索**: 门店/理发师/服务的搜索和地理距离查询，写入通过RabbitMQ异步同步
5. **Redis多层缓存策略**: 热点数据(门店/理发师/服务)本地缓存+Redis分布式缓存+CDN静态资源
6. **Casbin RBAC**: 基于角色的访问控制，支持数据权限（门店隔离/区域隔离）

### 数据流核心链路（预约下单）

```
用户选门店 → 选理发师 → 选服务 → 选时间 → 提交订单
    ↓
创建预约记录(MySQL事务) → 锁定时间段(Redis分布式锁) → 创建待支付订单
    ↓
调用微信支付 → RabbitMQ异步监听支付回调 → 更新订单状态 → 发送预约确认消息(WebSocket+微信模板消息) → 扣减库存(如有) → ES更新搜索索引
```

## 实现要点

1. **目录结构修复**: 当前apps目录下因shell brace expansion问题导致子目录结构异常（显示为`{src/`），需要重新正确创建完整目录树
2. **数据库设计先行**: 约40+张MySQL表，需要完整的ER设计和迁移脚本（GORM AutoMigrate + 手动SQL迁移）
3. **公共代码抽取**: 三个小程序共享 `@haircut/shared` 包（类型定义、API请求封装、通用组件、常量、工具函数）
4. **后端分层严格**: Handler → Service → Repository → Model，每层通过接口解耦，方便测试和后续拆分
5. **WebSocket连接管理**: 使用map+读写锁管理连接，支持按用户ID/角色/门店推送
6. **文件上传统一走MinIO**: 所有图片/视频统一上传至MinIO，返回URL存入各业务表+MongoDB元数据
7. **多租户预留**: 所有业务表增加`tenant_id`字段，中间件自动注入当前租户上下文

## 整体设计风格

采用 **Apple HIG + Notion极简 + 美团丽人** 融合风格，整体呈现高端、干净、专业的品牌形象。

**核心视觉语言**:

- **纯白底色** (#FFFFFF) 作为主背景，营造洁净专业感
- **品牌色点缀** 采用深玫瑰金 (#C8A882) 作为主色调，传达优雅与品质感
- **大圆角卡片** (16px-24px border-radius)，配合细微投影 (box-shadow: 0 2px 12px rgba(0,0,0,0.06))
- **充足留白**，信息层次通过字号(32/20/16/14/12px阶梯)和字重(600/500/400)区分
- **微动效**: 页面切换淡入(fade-in 200ms)、卡片hover轻微上浮(translateY -2px)、按钮点击缩放(scale 0.98)
- **骨架屏**: 所有列表页加载态使用 shimmer 动画骨架屏
- **暗黑模式**: 完整支持浅色/深色主题切换，深色模式以深灰(#1A1A1A)为底

## 页面设计规划（共6个核心界面系统）

### 1. 总部运营后台（admin-web）- 最复杂的界面系统

#### Dashboard运营仪表盘页面

- **顶部统计卡片区**: 4个KPI指标卡片（今日GMV/订单量/活跃用户/转化率），每个卡片含趋势箭头和环比数值，使用渐变图标背景
- **中部图表区**: 左侧营收折线图(7日趋势)，右侧订单来源饼图(小程序/官网/线下)，下方门店排行榜横向条形图
- **底部实时动态区**: 最近订单流水列表（实时滚动），右侧待办事项（待审核门店/待处理退款/低库存预警）
- **布局**: 响应式Grid布局，左侧固定Sidebar导航，顶部Breadcrumb

#### 门店管理页面

- **左侧**: 可折叠树形组织架构选择器（集团>区域>城市）
- **右侧主区域**: 顶部搜索栏+筛选器（状态/星级/城市）+ 视图切换（表格/卡片），下方门店数据表格（含缩略图/名称/地址/评分/营业状态/操作按钮）
- **弹窗**: 新增/编辑门店Modal表单（含地图坐标选择器、营业执照上传、营业时间设置）

#### 数据分析大屏页面

- **全屏沉浸式**: 深色主题，数据可视化为主
- **顶部**: 核心指标大数字展示（GMV/客单价/复购率/留存率），带数字滚动动画
- **中部左**: 地图热力图（全国门店分布与营收热度）
- **中部右**: 漏斗图（浏览→预约→下单→支付→复购转化漏斗）
- **底部**: 留存曲线图 + 门店TOP10排名柱状图

### 2. 用户端小程序（user-miniapp）核心页面

#### 首页

- **顶部**: 城市定位栏（带定位图标+下拉箭头）+ 搜索框
- **Banner区**: 轮播图（优惠活动/新品上架/品牌宣传），圆角卡片式，指示器为小圆点
- **快捷入口**: 8宫格图标入口（附近门店/明星理发师/热门套餐/新人礼包/积分商城/AI测脸/优惠券/我的订单）
- **今日推荐**: 横向滑动卡片（推荐门店+距离+评分标签）
- **明星理发师**: 横向滑动卡片（头像圆形/姓名/星级/擅长标签/预约按钮）
- **热门套餐**: 纵向双列瀑布流卡片（封面图+套餐名+原价划线+现价+已售数量）
- **底部TabBar**: 首页/门店/预约/我的

#### 门店详情页

- **顶部**: 门店封面大图（支持左右滑动多图）+ 返回按钮+分享按钮
- **信息卡片**: 门店名称+星级评分+人均价格标签，地址+电话+营业时间+停车信息（icon+文字排列）
- **Tab切换**: 服务项目 | 理发师团队 | 用户评价 | 门店环境
- **底部固定栏**: 收藏按钮 + 立即预约按钮（品牌色填充，圆角胶囊形）

#### 在线预约流程页面（多步骤）

- **步骤条**: 顶部进度指示（选门店 ✓ → 选理发师 → 选服务 → 选时间 → 确认支付）
- **选时间页**: 日历组件（可选日期高亮）+ 时间段网格（已被预约时段灰显不可选，可选用品牌色填充圆角块）
- **确认订单页**: 预约信息汇总卡（门店+理发师+服务+时间）+ 优惠券选择（可展开列表选券显示抵扣金额）+ 积分抵扣开关 + 价格明细（原价-优惠-积分抵扣=实付金额）

### 3. 官网（official-website）核心页面

#### 首页

- **Hero区**: 全宽品牌视频/高质量图片背景，叠加半透明遮罩，居中显示品牌Slogan + "探索门店"CTA按钮 + 向下滚动引导动画
- **品牌故事区**: 左右交错图文排版（左侧大图右侧文字描述，交替排列）
- **精选门店区**: 标题 + 3列卡片网格（门店封面+名称+城市+地址+查看详情链接）
- **明星理发师区**: 深色背景区分隔，横向滚动展示（圆形头像+姓名+职位+擅长风格标签）
- **数据展示区**: 4个大数字并排（全国门店数/注册会员数/年服务人次/满意度），数字计数动画
- **Footer**: 多列链接导航 + 微信公众号二维码 + 版权信息

### 4. 理发师端/店长端小程序

- 工作台首页采用卡片式Dashboard（数据卡片+快捷操作入口+今日待办列表）
- 整体与用户端保持一致的视觉语言，但功能更偏数据展示和操作效率

- Purpose: 在项目开发过程中进行大规模代码搜索、跨文件依赖分析、代码模式一致性检查
- Expected outcome: 快速定位相关代码、确保架构一致性、发现潜在问题

- Purpose: 将完整的项目设计文档集打包输出为PDF交付物
